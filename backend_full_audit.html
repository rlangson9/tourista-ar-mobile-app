🚨 Blockers — fix before any testing4 issues
blocker
Validation schemas wired nowhere except auth
validation.ts has 7 schemas. Only registerValidation + loginValidation are used. The other 5 (bookingValidation, messageValidation, tourValidation, productValidation, partnerValidation, supplierValidation) are defined but never imported anywhere.
blocker
No POST /api/orders route — B2B trade is dead
orderValidation is written, Order model exists, supplier can view orders — but there is no route to create one. The entire trade checkout flow has nowhere to land.
blocker
PUT /api/bookings/:id/status — no auth or ownership enforcement
The comment says "partner only" but the middleware stack is just protect. Any logged-in user can set any booking to cancelled or completed. Needs partner guard + a check that the booking's partnerId matches the requester.
blocker
apiLimiter and messageLimiter defined but never applied
apiLimiter (100 req/min) and messageLimiter (30 msg/min) are exported from rateLimit.ts but never imported into server.ts or the message route. Spam is completely unprotected.
⚠️ Warnings — fix before launch8 issues
warn
No database indexes on any model
Message queries by conversationId, senderId, receiverId on every send. Tour queries by location, category. Booking by userId, partnerId. Full collection scans from day one.
warn
Nodemailer transporter recreated on every single email call
Both auth.ts and admin.ts call nodemailer.createTransport() inline on each request. A new SMTP connection is opened every time. Should be a singleton module shared across the app.
warn
Users cannot cancel their own bookings
The status update route is partner-only by intent. There is no user-facing cancellation endpoint. Users who want to cancel are completely stuck.
warn
Rating fields exist on Tour and Product but no review route
rating and reviewCount are in both schemas. There is no POST /api/tours/:id/reviews or GET /api/tours/:id/reviews. Those fields will stay 0 forever.
warn
No full-text search on tours or products
Tour filtering supports category, location (regex), and price range. There is no keyword search on title or description. The ExploreScreen search bar will hit the API and get no results.
warn
tourValidation validates wrong field name
tourValidation checks body('destination') but the Tour model field is called location. The validation will always pass on this field and location will never be validated.
warn
Stripe API version is outdated
apiVersion: '2023-10-16' — current is 2024-06-20. Non-breaking now but Stripe deprecates old versions and will start sending warnings.
warn
Any user can read any conversation by ID
GET /api/messages/:conversationId fetches all messages in that conversation with no check that the requesting user is actually a participant. Anyone with a valid conversationId can read private messages.
